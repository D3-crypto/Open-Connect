# Next Steps: Building the Android Kotlin Engine

Now that the Desktop Rust Daemon is successfully capturing, encrypting, and broadcasting clipboard data, we need a "Target Sink" on the other side to receive it. Building the Android Kotlin Engine is the logical next step.

## Why Android Next?

1.  **Completing the Sync Loop:** Currently, the laptop broadcasts data into the void. By building the Android app, we can instantly test actual cross-device sync.
2.  **Validating Encryption:** We need to prove that Android's `Tink` or native `libsodium` bindings can successfully decrypt the Curve25519 payload generated by the Rust Daemon.
3.  **Tackling Android's Restrictions:** Android's background execution limits (Doze mode) are notoriously difficult. It's better to conquer the core networking and Foreground Service architecture now before layering a complex Flutter UI on top.

## Architecture of the Android Engine

We will build the core Android app using native Kotlin to ensure maximum performance and system-level access.

### Core Components:

1.  **Foreground Service (`SyncService.kt`):** The beating heart of the Android app. This runs persistently in the background (showing an ongoing notification) to prevent Android from killing our WebSocket connection.
2.  **Local Discovery (`NsdManager`):** Android's native implementation of mDNS. It will listen for the `_openconnect._tcp.local` broadcast coming from the Rust Daemon to find the laptop's IP address automatically.
3.  **WebSocket Client (`OkHttp`):** A robust client to maintain a persistent connection to the Rust Daemon's server.
4.  **Cryptography (`Tink` or `Kalium`):** The cryptographic engine required to generate the phone's identity and decrypt the incoming payloads.
5.  **Clipboard Injection (`ClipboardManager`):** The system hook to silently overwrite the Android clipboard when a trusted payload arrives.

### The Development Path

1.  Initialize the Android project structure (already scaffolded in `/mnt/ezio/OpenConnect/android`).
2.  Implement the `SyncService` to ensure background persistence.
3.  Implement the `NsdManager` to discover the laptop on the local Wi-Fi.
4.  Implement the `OkHttp` WebSocket client to connect and receive the JSON `Envelope`.
5.  Implement the decryption and clipboard injection logic.
